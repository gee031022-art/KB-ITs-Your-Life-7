1

HTML – 웹의 뼈대

HTML은 웹 페이지의 구조를 만드는 언어입니다. 브라우저는 이걸 받아서 화면을 렌더링합니다. 태그를 통해 제목, 문단, 이미지, 링크 등 모든 요소를 정의하죠.

// HTML 기본 구조
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>내 페이지</title>
  </head>
  <body>
    <h1>Hello World</h1>
  </body>
</html>

시멘틱 태그(Semantic Tag)는 HTML5에서 새로 생긴 개념입니다. 단순히 구조를 잡는 게 아니라 의미를 부여하는 것이 핵심입니다. 이게 왜 중요하냐면 유지보수, SEO, 웹 접근성 모두에 직결되기 때문입니다.

태그의미사용 위치
<header>상단 영역로고, 네비게이션
<nav>메뉴 영역링크 목록
<section>콘텐츠 구역본문 내 구분
<article>독립 콘텐츠게시글, 뉴스
<footer>하단 영역저작권, 연락처
🧠 HTML4까지는 <div>로 다 때웠는데, HTML5부터는 시멘틱 태그로 의미를 주는 것이 표준이 됐습니다. 수업 중 인상 깊었던 한마디 — "검색엔진도 사람처럼 읽는다"
2

CSS – 디자인

CSS는 HTML에 스타일을 입히는 언어입니다. 선택자(Selector)로 대상을 지정하고, 속성(Property)으로 꾸밉니다. 수업에서 가장 많이 나온 개념은 박스 모델과 Flexbox였습니다.

📦 박스 모델 (Box Model)

모든 HTML 요소는 박스 형태입니다. 이걸 이해 못 하면 레이아웃이 계속 틀어집니다.

📄

content

실제 내용 영역. width/height로 조절

🔲

padding

안쪽 여백. 배경색이 적용됨

📏

border

테두리. 두께/스타일/색상 설정

↔

margin

바깥 여백. 요소 간 간격

// CSS 박스 모델 예제
.box {
  width: 200px;
  padding: 10px;
  border: 5px solid #333;
  margin: 20px;
  /* 실제 점유 너비 = 200 + 10×2 + 5×2 + 20×2 = 270px */
}

⚡ Flexbox — 레이아웃 끝판왕

float 시대는 지났습니다. 요즘 레이아웃은 Flexbox가 핵심입니다. 수평 정렬, 수직 정렬, 간격 배분을 단 몇 줄로 해결합니다.

// Flexbox 핵심 속성
.container {
  display: flex;
  justify-content: space-between; /* 가로 정렬 */
  align-items: center;         /* 세로 정렬 */
  gap: 20px;                   /* 아이템 간격 */
}
😵 CSS는 이론은 이해해도 직접 코딩하면 왜 이렇게 안 되는 건지... position의 absolute / relative / fixed 차이가 제일 헷갈렸습니다. 특히 자식이 absolute면 가장 가까운 포지셔닝된 부모를 기준으로 움직인다는 개념을 여러 번 반복해야 했어요.
3

JavaScript – 동작

JavaScript는 웹을 "살아있게" 만드는 언어입니다. 클릭, 입력, 계산, 화면 변경 등 모든 동적 동작을 담당합니다. 다른 언어와 가장 다른 점은 함수도 값이라는 것입니다.

개념설명예시
== vs ===값만 비교 vs 타입까지 비교'10'==10 → true, '10'===10 → false
let / const블록 스코프 변수 / 상수const PI = 3.14;
var함수 스코프 (권장 X)호이스팅 문제 있음
callback함수를 인수로 전달addEventListener("click", fn)
for...of값 순회값이 변수에 전달됨
for...in키 순회인덱스/키가 변수에 전달됨

🔑 콜백 함수 (Callback) — 가장 중요한 개념

수업에서 가장 강조한 부분입니다. 함수를 매개변수로 전달할 수 있다는 것, 이게 이벤트 처리의 핵심입니다.

// 콜백 함수 — 이벤트 처리의 핵심
// 함수도 값이다 → 변수에 저장 가능
let sayHello = function() {
  alert('안녕하세요!');
};

// 함수를 매개변수로 전달 → 이게 콜백!
document.querySelector('button')
  .addEventListener('click', sayHello);

// 또는 익명 함수로 바로 전달
document.querySelector('button')
  .addEventListener('click', function() {
    console.log('클릭됨');
  });
🤯 솔직히 콜백이랑 클로저(closure) 개념에서 완전히 멈췄습니다. "함수가 자신이 만들어진 환경을 기억한다"는 게 이론으론 이해해도 코드로 보면 아직 낯선 느낌... 이건 더 써봐야 할 것 같습니다.

웹 동작 원리 – HTTP 통신

코드만 짜다 보면 놓치기 쉬운 부분인데, 웹이 어떻게 동작하는지 기본 원리를 이해해야 나중에 백엔드랑 연결할 때 막히지 않습니다.

HTTP 통신 흐름
사용자 요청 → 브라우저 → 서버 → 응답 → 화면 출력
👉 connect → request → response → close (stateless)
구분GETPOST
데이터 위치URL에 포함Body에 포함
보안낮음 (URL 노출)상대적으로 안전
용도데이터 조회데이터 전송

보안 관점 – 이건 진짜 중요합니다

수업 중간중간 "보안 이슈?"라는 메모가 자꾸 등장했습니다. 웹을 만들면서 동시에 어떻게 공격당하는지도 생각해야 합니다. 보안 전공이라 이 부분이 특히 흥미로웠습니다.

🔐 수업에서 나온 주요 보안 이슈

  • 파라미터 조작 — ?price=10000 → ?price=100 으로 변경 가능
  • 쿠키 조작 — role=user → role=admin 으로 변경 시도
  • URL 조작 — /user?id=1 → /user?id=2 로 타인 정보 접근
  • 절대경로 노출 — 서버 구조가 드러나 취약점이 될 수 있음
  • 불필요한 포트 오픈 — 공격자의 스캐닝 대상이 됨
  • 파일 업로드 취약점 — 악성 스크립트(웹쉘) 업로드 가능
핵심 원칙 — "클라이언트는 절대 신뢰하지 말 것"
클라이언트가 보내는 모든 값(URL, 파라미터, 쿠키, 헤더)은 조작 가능합니다.
👉 서버 측 검증이 반드시 필요합니다
6

실습 – 스타벅스 클론 느낌

이론만 하면 의미 없죠. 수업 실습으로 간단한 레이아웃을 만들어봤습니다. 상단 고정 네비게이션, 사이드바, 스크롤 가능한 본문, 하단 푸터 구조입니다.

// 스타벅스 클론 핵심 CSS 구조
.top_bar {
  position: fixed;   /* 스크롤해도 고정 */
  top: 0; left: 0; right: 0;
  display: flex;
  justify-content: space-between;
}

.main {
  position: absolute;
  top: 100px;       /* 상단바 높이만큼 */
  overflow-y: scroll;
}

.footer {
  position: fixed;   /* 하단도 고정 */
  bottom: 0;
}
💡 position을 이해하니까 레이아웃이 드디어 보이기 시작했습니다. fixed는 뷰포트 기준, absolute는 포지셔닝된 부모 기준, relative는 자기 원래 위치 기준 — 이 3개를 구분하는 것이 핵심이었습니다.
7

솔직한 후기

CSS는 생각보다 훨씬 어렵다. 이론으로 이해하는 것과 직접 짜는 것은 완전히 달랐습니다. position, float, flex 이 셋이 뒤섞이면 화면이 예상과 전혀 다르게 나오는 게 일상이었습니다.

JavaScript는 개념이 중요하다는 걸 느꼈습니다. 특히 콜백 함수와 스코프 개념은 코드를 보면서 머릿속으로 흐름을 그려야 이해가 됩니다. 아직 클로저는 완전히 납득되지 않아서 더 써봐야 할 것 같습니다.

HTTP 통신 부분이 의외로 재밌었습니다. 보안 전공이라 보안 이슈가 나올 때마다 메모했는데, 나중에 백엔드 배울 때 다시 연결될 것 같습니다. 웹은 정말 구조 + 디자인 + 동작 + 통신 + 보안이 모두 맞물려야 한다는 걸 이번 주차에서 제대로 느꼈습니다.
